# Implementation Plan

- [ ] 1. DB層: 全チャンネル横断の動画取得メソッド
- [ ] 1.1 (P) 全動画を配信日時降順で取得するDBメソッドを追加する
  - チャンネルIDを問わず全動画を対象とする
  - `COALESCE(broadcast_start_at, published_at) DESC`でソートする
  - `count`パラメータで取得件数を制限する
  - 0件時は空リストを返す
  - _Requirements: 1.1, 1.3, 1.4_

- [ ] 1.2 (P) DBメソッドのユニットテストを追加する
  - 件数制限が正しく機能することを検証する
  - ソート順が配信日時降順であることを検証する
  - 0件時に空リストを返すことを検証する
  - 複数チャンネルの動画が混在するケースを検証する
  - _Requirements: 1.1, 1.3, 1.4, 1.5_

- [ ] 2. TUI層: 単一選択メニュー関数
- [ ] 2.1 (P) 一覧から1つだけ選択するTUIヘルパー関数を追加する
  - beaupyの`select`関数を使用して単一選択メニューを表示する
  - ターミナル高さに基づいてページサイズを算出する
  - キャンセル時（Ctrl+C/Esc）はNoneを返す
  - _Requirements: 2.1, 2.5_

- [ ] 3. CLI層: videosコマンドの実装
- [ ] 3.1 videosコマンドの通常モードを実装する
  - `@cli.command()`として`videos`コマンドを登録する
  - `--count`オプション（デフォルト20）で表示件数を制御する
  - 各動画について配信日時、タイトル、YouTube URLを表示する
  - 動画が0件の場合はメッセージを表示して終了する
  - _Requirements: 1.1, 1.2, 1.3, 1.4, 1.5_

- [ ] 3.2 videosコマンドのTUIモードを実装する
  - `--tui`フラグで動画一覧を単一選択メニューで表示する
  - 動画選択後にsegments/suggestの操作選択メニューを表示する
  - segments選択時は既存の`_run_tui_flow_segments`を呼び出す
  - suggest選択時は既存の`_run_tui_flow_suggest`相当の処理を実行する
  - キャンセル時はメッセージを表示して終了する
  - _Requirements: 2.1, 2.2, 2.3, 2.4, 2.5_

- [ ] 3.3 videosコマンドのテストを追加する
  - 通常モードの出力フォーマットを検証する
  - 0件時のメッセージ出力を検証する
  - `--count`オプションによる件数制限を検証する
  - _Requirements: 1.1, 1.2, 1.3, 1.4, 1.5_
