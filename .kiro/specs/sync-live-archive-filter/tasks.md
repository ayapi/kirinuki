# Implementation Plan

- [x] 1. ドメインモデルとインフラ層の拡張
- [x] 1.1 (P) スキップ理由と同期結果モデルに非ライブアーカイブ用の値を追加する
  - スキップ理由の列挙型に「ライブ配信アーカイブではない」を表す値を追加する
  - 同期結果モデルに、非ライブ動画としてスキップされた件数を集計するフィールドを追加する
  - チャンネル単位の同期結果集約時に、非ライブスキップ件数も合算されるようにする
  - _Requirements: 2.1, 2.2_

- [x] 1.2 (P) メタデータ取得処理にライブ配信状態の情報を含める
  - 動画メタデータのデータクラスに、ライブ配信状態を保持するフィールドを追加する（値がない場合は `None`）
  - メタデータ取得処理で、yt-dlpの抽出結果からライブ配信状態を読み取り、メタデータに含める
  - 追加のAPI呼び出しは行わず、既存の抽出処理から値を取得する
  - _Requirements: 1.2, 3.2_

- [x] 2. 同期処理にライブ配信アーカイブ判定とスキップロジックを追加する
  - メタデータ取得の直後、字幕取得の前にライブ配信状態を判定する
  - ライブ配信状態が「配信済みアーカイブ」の場合のみ字幕取得・保存処理に進む
  - ライブ配信状態が「配信済みアーカイブ」以外の場合、字幕取得をスキップし、スキップカウントとスキップ理由を更新する
  - ライブ配信状態が不明（`None`）の場合は安全側に倒してスキップせずにsync対象として処理する
  - スキップ時は動画ID、タイトル、ライブ配信状態をログに出力する
  - _Requirements: 1.1, 1.3, 1.4, 2.1, 2.3, 3.1_

- [x] 3. CLIの同期サマリーに非ライブスキップ件数の表示を追加する
  - スキップ理由のラベルマップに「配信アーカイブ以外」を追加する
  - 非ライブスキップ件数が結果サマリーの内訳に含まれるようにする
  - _Requirements: 2.2_

- [x] 4. テスト
- [x] 4.1 (P) メタデータ取得とライブ状態フィルタリングのユニットテストを追加する
  - メタデータ取得が `live_status` を正しく返すことを検証する（`was_live`, `not_live`, `None` の各ケース）
  - ライブ配信アーカイブ（`was_live`）の場合に字幕取得・保存が実行されることを検証する
  - 非ライブ動画（`not_live` 等）の場合にスキップされ、カウントとスキップ理由が更新されることを検証する
  - ライブ配信状態が不明（`None`）の場合にスキップせずsync対象として処理されることを検証する
  - _Requirements: 1.1, 1.2, 1.3, 1.4, 2.1, 2.3, 3.1, 3.2_

- [x] 4.2 (P) 同期パイプライン全体のインテグレーションテストを追加する
  - ライブ配信アーカイブと通常動画が混在するチャンネルで、ライブアーカイブのみがsyncされることを検証する
  - 非ライブスキップ件数とスキップ理由が同期結果に正しく集計されることを検証する
  - CLIの同期サマリー出力に非ライブスキップ件数が含まれることを検証する
  - _Requirements: 1.3, 1.4, 2.1, 2.2_
