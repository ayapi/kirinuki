# Implementation Plan

- [x] 1. 基盤セットアップ
- [x] 1.1 simple-term-menu依存追加とClipCandidateモデル定義
  - simple-term-menuパッケージをプロジェクト依存に追加する
  - TUI表示と切り抜き実行に必要な共通データを保持するClipCandidateモデルを定義する
  - 必須フィールド: 動画ID、開始・終了時間（ミリ秒）、話題要約、表示用ラベル
  - オプションフィールド: 動画タイトル、チャンネル名、検索スコア、マッチ種別、推薦スコア、魅力ポイント
  - _Requirements: 2.1, 3.1, 4.1, 5.1_

- [x] 1.2 (P) ファイル名自動生成ユーティリティの実装とテスト
  - ファイル名に使用できない文字（`/ \ : * ? " < > |` および制御文字）を除去するサニタイズ関数を実装する
  - 空白をアンダースコアに置換し、末尾のドット・スペースを除去する
  - 話題名を最大50文字に切り詰める処理を含める
  - 動画ID・開始時間・話題名から `{動画ID}-{M}m{SS}s-{サニタイズ済み話題名}.mp4` 形式のファイル名を生成する関数を実装する
  - 開始時間は総分数表記（例: 18m03s, 72m15s）
  - ユニットテスト: 通常ケース、特殊文字を含む話題名、長い話題名の切り詰め、1時間超の開始時間、エッジケース（0秒、空白のみの話題名）
  - _Requirements: 5.1, 5.2, 5.3, 5.4_

- [x] 2. TUIコアコンポーネント
- [x] 2.1 (P) アダプター関数の実装とテスト
  - SearchResult一覧をClipCandidate一覧に変換する関数を実装する。SearchResult.youtube_urlから既存のextract_video_id()でvideo_idを取得する。表示ラベルには検索スコア、マッチ種別、動画タイトル、時間範囲、話題要約を含める
  - Segment一覧をClipCandidate一覧に変換する関数を実装する。表示ラベルには時間範囲と話題要約を含める
  - SuggestResult（VideoWithRecommendationsのリスト）をClipCandidate一覧に変換する関数を実装する。SegmentRecommendationの時間（秒単位）をミリ秒に変換する。表示ラベルには推薦スコア、動画タイトル、時間範囲、話題要約を含める
  - video_id抽出に失敗したSearchResult項目はスキップする
  - 時間範囲のフォーマットには既存のformat_time_range()を活用する
  - ユニットテスト: 各変換の正確性、display_labelのフォーマット検証、空リスト入力、video_id抽出失敗時のスキップ
  - _Requirements: 2.1, 2.2, 2.3, 2.4_

- [x] 2.2 (P) TUIマルチセレクトメニューの実装
  - simple-term-menuのTerminalMenuを使用してClipCandidate一覧をマルチセレクトメニューとして表示する関数を実装する
  - Spaceキーで選択状態をトグルし、選択済み項目にチェックマーク等の視覚的マーカーを表示する
  - 上下矢印キーでカーソル移動を可能にする
  - ステータスバーに現在の選択件数を表示する
  - 1件以上選択時にEnterで確定可能であることをユーザーに示す
  - q/Escキーでキャンセル時は空リストを返す
  - Ctrl+C時も安全にTUI画面を終了する
  - キャンセル・選択なしの場合の戻り値を空リストとして統一する
  - _Requirements: 2.5, 3.1, 3.2, 3.3, 3.4, 6.1, 6.2_

- [x] 3. 切り抜き実行とCLI統合
- [x] 3.1 TUIクリップ実行機能の実装
  - 選択されたClipCandidate一覧を受け取り、順次切り抜きを実行する関数を実装する
  - 各候補に対してファイル名自動生成関数でファイル名を作成し、既存のClipServiceを1レンジのMultiClipRequestとして呼び出す
  - 処理進捗を「N件目/全M件 切り抜き中...」の形式で表示する
  - 完了後に成功件数・失敗件数と各出力ファイルパスのサマリーを表示する
  - セグメントが1件も選択されていない場合はその旨を通知して切り抜きを実行しない
  - 出力先ディレクトリとして既存のデフォルト出力先（`~/.kirinuki/output`）を使用する
  - Ctrl+Cによる中断時は現在処理中のセグメント完了後に残りをスキップし、それまでの結果をサマリー表示する
  - ClipService生成の共通ヘルパーを用意し、各コマンドからの利用を統一する
  - _Requirements: 4.1, 4.2, 4.3, 4.4, 5.5, 6.3_

- [x] 3.2 search/segments/suggestコマンドへの--tuiフラグ統合
  - searchコマンドに`--tui`フラグを追加し、指定時はサービス結果をアダプター→TUIセレクター→クリップ実行の流れで処理する
  - segmentsコマンドに`--tui`フラグを追加し、同様の流れで処理する
  - suggestコマンドに`--tui`フラグを追加し、同様の流れで処理する（suggestは独自のサービス初期化パスを使用）
  - `--tui`フラグが指定されていない場合は従来のテキスト出力をそのまま維持する（後方互換性）
  - 結果が0件の場合はTUIを起動せず「結果がありません」と表示して終了する
  - パイプやリダイレクト先でのターミナル非対応時のエラーハンドリングを実装する
  - _Requirements: 1.1, 1.2, 1.3, 1.4_
