# Requirements Document

## Introduction

`search`コマンドに動画IDによる絞り込み検索オプションを追加する。現在の`search`コマンドはキーワード検索（FTS5）とセマンティック検索（ベクトル）のハイブリッド検索を全動画横断で実行するが、特定の動画に限定して検索したいユースケースに対応していない。本機能により、ユーザーは`--video-id`オプションで1つまたは複数の動画IDを指定し、対象動画のセグメントのみに絞り込んで検索できるようになる。

## Requirements

### Requirement 1: 動画IDフィルタオプション

**Objective:** ユーザーとして、`search`コマンドに動画IDを指定して検索対象を絞り込みたい。特定の配信内で話していた内容を効率的に見つけるため。

#### Acceptance Criteria
1. When ユーザーが`search`コマンドに`--video-id`オプションを指定した場合, the CLIは指定された動画IDに属するセグメントのみを検索対象とする
2. When ユーザーが`--video-id`オプションを複数回指定した場合, the CLIは指定されたすべての動画IDに属するセグメントを検索対象とする（OR条件）
3. When `--video-id`オプションが省略された場合, the CLIは従来通り全動画を対象に検索を実行する（後方互換性を維持）
4. If 指定された動画IDがデータベースに存在しない場合, the CLIは該当する動画IDが見つからない旨を警告として表示し、存在する動画IDのみで検索を続行する
5. If 指定されたすべての動画IDがデータベースに存在しない場合, the CLIは検索結果が0件である旨を表示し正常終了する

### Requirement 2: キーワード検索（FTS）での動画IDフィルタ適用

**Objective:** ユーザーとして、動画IDフィルタがキーワード検索に正しく適用されることで、指定動画内のテキスト一致結果のみを取得したい。

#### Acceptance Criteria
1. When `--video-id`オプションとともにキーワード検索が実行された場合, the 検索サービスは指定動画IDに紐づくセグメントのFTS結果のみを返す
2. While 動画IDフィルタが適用されている状態で, the FTS検索のスニペット生成・スコアリングは従来と同じロジックで動作する

### Requirement 3: セマンティック検索（ベクトル）での動画IDフィルタ適用

**Objective:** ユーザーとして、動画IDフィルタがセマンティック検索にも正しく適用されることで、指定動画内の意味的に近い結果のみを取得したい。

#### Acceptance Criteria
1. When `--video-id`オプションとともにセマンティック検索が実行された場合, the 検索サービスは指定動画IDに紐づくセグメントのベクトル検索結果のみを返す
2. While 動画IDフィルタが適用されている状態で, the ベクトル検索のスコアリング・ランキングは従来と同じロジックで動作する

### Requirement 4: TUIモードとの統合

**Objective:** ユーザーとして、`--tui`モードでも動画IDフィルタが機能し、絞り込んだ結果からインタラクティブに切り抜き対象を選択したい。

#### Acceptance Criteria
1. When `--video-id`と`--tui`の両オプションが同時に指定された場合, the CLIはフィルタ適用後の検索結果をTUIセレクターに渡す
2. While TUIモードで動画IDフィルタが適用されている状態で, the 選択・切り抜き実行フローは従来と同じ動作をする
