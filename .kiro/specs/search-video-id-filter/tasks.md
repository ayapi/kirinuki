# Implementation Plan

- [x] 1. Database層: video_idフィルタ対応
- [x] 1.1 (P) 動画ID存在確認機能の追加
  - 指定されたvideo_idリストをDBに問い合わせ、存在するIDと存在しないIDに分離する
  - 一括クエリで効率的に確認し、結果をタプルで返す
  - _Requirements: 1.4, 1.5_

- [x] 1.2 (P) FTS検索への動画IDフィルタ条件の追加
  - FTSセグメント検索にオプショナルなvideo_idリストを受け取る引数を追加する
  - video_idが指定された場合、セグメントのvideo_idがリスト内のいずれかと一致するもののみを返すようSQLを動的に構築する
  - video_idが未指定の場合は既存のクエリをそのまま実行し、後方互換性を維持する
  - スニペット生成・スコアリングのロジックは変更しない
  - _Requirements: 2.1, 2.2, 1.3_

- [x] 1.3 (P) ベクトル検索への動画IDフィルタ適用
  - ベクトル検索にオプショナルなvideo_idリストを受け取る引数を追加する
  - video_idが指定された場合、KNN検索のk値をオーバーフェッチ倍率（5倍）で拡大して取得し、JOIN後にvideo_idでフィルタリングしてからlimit件に切り詰める
  - sqlite-vecのvec0テーブルはKNN検索時にWHERE条件を直接適用できないため、post-filter方式を採用する
  - video_idが未指定の場合は既存のクエリをそのまま実行し、後方互換性を維持する
  - スコアリング・ランキングのロジックは変更しない
  - _Requirements: 3.1, 3.2, 1.3_

- [x] 2. SearchServiceのvideo_idsフィルタ対応
  - 検索メソッドにオプショナルなvideo_idsパラメータを追加する
  - video_idsが指定された場合、まずDB側の存在確認を呼び出し、存在しないIDに対する警告メッセージを生成する
  - 存在するIDのみをFTS検索・ベクトル検索に伝播し、フィルタ付きで実行する
  - 全IDが不在の場合は空の結果リストと警告を返す
  - 戻り値をタプル（検索結果リスト、警告メッセージリスト）に変更する
  - video_idsが未指定の場合は従来どおりの動作で、警告は空リストを返す
  - 既存のマージ・スコアリングロジックは変更しない
  - _Requirements: 1.2, 1.3, 1.4, 1.5, 2.1, 3.1_

- [x] 3. CLIの--video-idオプション追加とUI統合
  - searchコマンドに`--video-id`オプション（複数回指定可）を追加する
  - 空タプルの場合はフィルタなし（None）としてSearchServiceに渡す
  - SearchServiceの戻り値変更に合わせ、警告メッセージがあれば結果表示前に出力する
  - 既存の結果表示フローとTUIフローへの受け渡しは変更しない（フィルタ適用後の結果がそのまま渡される）
  - _Requirements: 1.1, 1.2, 1.3, 1.4, 1.5, 4.1, 4.2_

- [x] 4. テスト
- [x] 4.1 (P) Database層のvideo_idフィルタテスト
  - video_id存在確認が正しく存在/不在を分離することを確認する
  - FTS検索でvideo_idフィルタ指定時に対象動画のセグメントのみ返されることを確認する
  - FTS検索でvideo_id未指定時に全セグメントが返されることを確認する（後方互換）
  - ベクトル検索でvideo_idフィルタ指定時に対象動画のセグメントのみ返されることを確認する
  - ベクトル検索でvideo_id未指定時に全セグメントが返されることを確認する（後方互換）
  - _Requirements: 1.3, 1.4, 2.1, 2.2, 3.1, 3.2_

- [x] 4.2 (P) SearchServiceとCLIのテスト
  - video_ids指定時に正しい結果と警告が返ることを確認する
  - 存在しないvideo_id指定時に警告メッセージが生成されることを確認する
  - 全video_idが不在の場合に空結果と警告が返ることを確認する
  - video_ids未指定時に従来どおりの動作であることを確認する
  - CLIから--video-idオプションがSearchServiceに正しく伝播されることを確認する
  - _Requirements: 1.1, 1.2, 1.3, 1.4, 1.5, 4.1, 4.2_
