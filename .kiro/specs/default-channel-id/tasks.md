# Implementation Plan

- [x] 1. チャンネルID自動解決関数の実装
- [x] 1.1 チャンネルIDの自動解決ロジックを実装する
  - チャンネルIDが指定されている場合はそのまま返却する
  - チャンネルIDが省略された場合、登録チャンネル一覧を取得して件数で分岐する
  - 登録チャンネルが1つの場合、そのチャンネルIDを返却し、使用チャンネルの情報を標準エラー出力に通知する
  - 登録チャンネルが0件の場合、チャンネル未登録の旨とチャンネル登録コマンドの案内を含むエラーを返す
  - 登録チャンネルが複数の場合、チャンネルIDの指定が必要な旨と登録チャンネル一覧（名前とID）を含むエラーを返す
  - _Requirements: 1.1, 1.2, 1.3, 1.4, 2.3, 3.1, 3.2_

- [x] 1.2 チャンネルID自動解決のユニットテストを作成する
  - チャンネルIDが明示指定された場合にそのまま返却されることを検証する
  - 登録チャンネルが1つの場合にそのIDが返却されることを検証する
  - 登録チャンネルが1つの場合に標準エラー出力に通知メッセージが出力されることを検証する
  - 登録チャンネルが0件の場合にエラーが発生することを検証する
  - 登録チャンネルが複数の場合にエラーが発生し、エラーメッセージにチャンネル一覧が含まれることを検証する
  - _Requirements: 1.1, 1.2, 1.3, 1.4, 3.1, 3.2_

- [x] 2. 対象コマンドへの統合
- [x] 2.1 (P) `channel videos` コマンドのチャンネルID引数をオプショナルに変更する
  - チャンネルIDの位置引数を省略可能に変更する
  - コマンド実行時にチャンネルID自動解決関数を呼び出して、省略時の解決を行う
  - 明示指定時は既存の動作を維持する
  - _Requirements: 1.4, 2.1_

- [x] 2.2 (P) `suggest` コマンドのチャンネルID引数をオプショナルに変更する
  - チャンネルIDの位置引数を省略可能に変更する
  - コマンド実行時にチャンネルID自動解決関数を呼び出して、省略時の解決を行う
  - `suggest` コマンドは独自のDB接続パターンを使用しているため、解決関数が利用するDBインスタンスの生成が必要
  - 明示指定時は既存の動作を維持する
  - _Requirements: 1.4, 2.2_

- [x] 3. 統合テスト
- [x] 3.1 コマンド統合テストを作成する
  - `channel videos` コマンドを1チャンネル登録状態でチャンネルID省略実行し、正常動作を検証する
  - `suggest` コマンドを1チャンネル登録状態でチャンネルID省略実行し、正常動作を検証する
  - 複数チャンネル登録状態でチャンネルID省略実行し、エラーメッセージとチャンネル一覧の表示を検証する
  - チャンネルID明示指定時に従来通り動作することを検証する
  - _Requirements: 1.1, 1.2, 1.4, 2.1, 2.2, 3.1, 3.2_
