# Implementation Plan

- [x] 1. 基盤コンポーネントの実装
- [x] 1.1 (P) 例外クラス階層を定義する
  - 基底例外`SegmentExtractorError`と、それを継承する具象例外クラスを定義する
  - 無効URL、時間範囲不正、認証必要、ダウンロード失敗、ffmpeg未検出、切り出し失敗の6種類を用意する
  - 各例外は元のエラーメッセージや原因を保持できるようにする
  - _Requirements: 4.3, 5.1, 5.2, 5.3, 5.4, 5.5_

- [x] 1.2 (P) 切り抜きリクエスト・結果のデータモデルを定義する
  - 切り抜きリクエストモデルを定義する: URL、開始秒・終了秒（省略可）、出力先パス（省略可）、出力フォーマット（デフォルトmp4）、Cookieファイルパス（省略可）、一時ディレクトリ（省略可）
  - 開始秒・終了秒の少なくとも一方が指定されていることをバリデーションする
  - 両方指定時に開始秒 < 終了秒であることを検証する
  - 開始秒が0以上であること、出力フォーマットがサポート対象（mp4, mkv, webm）であることを検証する
  - 切り抜き結果モデルを定義する: 出力パス、動画ID、開始秒、終了秒、切り抜き時間長
  - _Requirements: 1.2, 1.3, 1.4, 1.5, 3.1, 3.2, 3.3, 3.4, 5.3, 6.2, 6.3_

- [x] 1.3 (P) ユーティリティ関数を実装する
  - YouTube URLから動画IDを抽出する関数を実装する（`youtube.com/watch?v=`、`youtu.be/`、`youtube.com/live/`等の各形式に対応）
  - 無効なURLの場合は適切な例外を送出する
  - 秒数（float）を`HH:MM:SS.mmm`形式に変換する関数を実装する（ffmpegコマンド生成用）
  - `HH:MM:SS`形式の文字列を秒数（float）に変換する関数を実装する（外部からの入力変換用）
  - 動画IDと時間範囲からデフォルトの出力ファイル名を生成する関数を実装する
  - _Requirements: 1.4, 3.2, 5.1_

- [x] 2. インフラ層の実装
- [x] 2.1 (P) ffmpegによる動画区間切り出し機能を実装する
  - ffmpegがシステムにインストールされているかを確認する機能を実装する（未検出時は専用例外を送出）
  - 入力動画ファイル・出力パス・開始秒・終了秒を受け取り、ffmpegで区間を切り出す機能を実装する
  - ffmpegコマンドは`-ss`を`-i`の前に配置（キーフレームシーク）し、`-c copy`でストリームコピーを行う
  - ffmpegプロセスの失敗時はstderrの内容を含む例外を送出する
  - Protocol定義に従い、テスト時にモック差し替え可能なインターフェースとする
  - _Requirements: 1.1, 5.4_

- [x] 2.2 (P) yt-dlpによる動画ダウンロード機能を実装する
  - 動画IDと出力先ディレクトリを受け取り、yt-dlp Python APIで動画をダウンロードする機能を実装する
  - ダウンロードフォーマットは`bestvideo[ext=mp4]+bestaudio[ext=m4a]/best[ext=mp4]`をデフォルトとする
  - Cookieファイルパスが指定されている場合、yt-dlpの`cookiefile`オプションに渡してメンバー限定動画に対応する
  - ダウンロード完了後、実際のファイルパスを返却する
  - yt-dlpのダウンロードエラーおよび認証エラーを適切な例外クラスにマッピングする
  - 既存のYtdlpClient Protocolに追加メソッドとして統合する
  - _Requirements: 2.1, 4.1, 4.2, 4.3, 5.5_

- [x] 3. SegmentExtractorServiceを実装する
  - 切り抜きリクエストを受け取り、DL→区間切り出し→クリーンアップの一連のパイプラインをオーケストレーションする
  - 処理開始前にURLの妥当性検証と動画メタデータ取得による時間範囲の検証を行い、不正な入力でのダウンロードを防止する
  - 一時ディレクトリをコンテキストマネージャで管理し、正常終了・例外発生いずれの場合もダウンロード済み元動画を確実に削除する
  - 出力先パスが未指定の場合、カレントディレクトリに動画ID・時間範囲を含むファイル名を自動生成する
  - 開始時刻のみ指定時は動画末尾まで、終了時刻のみ指定時は動画冒頭からの切り出しを行う
  - インフラ層のProtocol（動画ダウンロード、ffmpeg切り出し）への依存はコンストラクタ注入で受け取り、コア層から外部ツールを直接importしない
  - _Requirements: 1.1, 1.2, 1.3, 1.5, 2.1, 2.2, 2.3, 2.4, 3.2, 4.3, 5.1, 5.2, 5.5, 6.1, 6.4_

- [x] 4. テストの実装
- [x] 4.1 (P) ユニットテストを実装する
  - 切り抜きリクエストモデルのバリデーションテスト: 正常ケース、開始>終了の拒否、両方未指定の拒否、サポート外フォーマットの拒否
  - YouTube URL解析テスト: 各URL形式（watch?v=, youtu.be/, live/）からの動画ID抽出、無効URLの例外送出
  - 時刻変換テスト: 秒数→HH:MM:SS.mmm変換の正確性、HH:MM:SS→秒数変換
  - デフォルト出力ファイル名生成テスト: 動画ID・時間範囲を含むファイル名フォーマットの検証
  - _Requirements: 1.2, 1.3, 1.4, 3.2, 5.1, 5.3, 6.2_

- [x] 4.2 インテグレーションテストを実装する
  - FfmpegClient: ffmpeg存在確認（モック時の成功/未検出）、区間切り出しコマンドの正常実行とエラー時の例外送出をモック化してテスト
  - YtdlpClient動画ダウンロード: ダウンロード成功・失敗・認証エラーのケースをyt-dlp APIモックでテスト
  - SegmentExtractorService統合テスト: 全依存をモック化し、DL→切り出し→クリーンアップの一連のフローを検証する
  - 一時ファイルクリーンアップ: 正常終了時および例外発生時に一時ディレクトリが確実に削除されることを検証する
  - 開始時刻のみ指定、終了時刻のみ指定、出力パス自動生成の各パターンを検証する
  - _Requirements: 1.1, 1.2, 1.3, 2.1, 2.2, 2.3, 5.4, 5.5_
