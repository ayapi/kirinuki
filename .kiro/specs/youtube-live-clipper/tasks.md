# Implementation Plan

- [x] 1. プロジェクトセットアップと基盤構築
- [x] 1.1 Pythonプロジェクト初期化
  - uvでプロジェクトを初期化し、`src/kirinuki/`をパッケージルートとするsrcレイアウトを構成する
  - `pyproject.toml`にプロジェクトメタデータと全依存ライブラリ（yt-dlp, anthropic, openai, sqlite-vec, click or typer, pydantic）を定義する
  - `src/kirinuki/`配下にcli, core, infra, modelsの各サブパッケージディレクトリを作成する
  - Ruff（linter/formatter）とpytest、mypyの開発用依存とその設定をpyproject.tomlに追加する
  - `uv sync`で依存を解決し、`uv run python -c "import kirinuki"`で基本的なパッケージインポートが動作することを確認する
  - _Requirements: 6.1_

- [x] 1.2 Pydanticモデルと設定管理
  - アプリケーション全体で使用するドメインモデル（チャンネル、動画、字幕行、セグメント、検索結果）をPydanticモデルとして定義する
  - アプリ設定モデルを作成し、LLM APIキー、Embedding APIキー、Cookieファイルパス、DBファイルパス、LLMモデル名をフィールドとして持たせる
  - 設定の読み込み優先順位を実装する: 環境変数 → 設定ファイル（`~/.kirinuki/config.toml`） → デフォルト値
  - 設定ファイルが存在しない場合はデフォルト値で動作し、初回実行時に設定ディレクトリを自動作成する
  - _Requirements: 5.1, 6.2_

- [x] 1.3 データベーススキーマとマイグレーション
  - 設計のPhysical Data Modelに基づき、channels, videos, subtitle_lines, segments テーブルを作成するスキーマを定義する
  - FTS5仮想テーブル（trigramトークナイザー）を作成し、字幕テキストの全文検索インデックスを構成する
  - sqlite-vecをロードし、セグメントベクトル用の仮想テーブルを作成する
  - スキーマバージョン管理テーブルを導入し、DB初期化時にバージョンチェックとマイグレーションを実行する仕組みを整える
  - WALモードを有効にしてDB接続を初期化する
  - チャンネル・動画・字幕行・セグメントに対する基本的なCRUD操作（挿入、取得、一覧、存在確認）を実装する
  - 字幕行を挿入する際にFTSテーブルにも同時に書き込む処理を実装する
  - セグメントとそのベクトルを一括で保存する処理を実装する
  - _Requirements: 2.1, 2.2, 2.3_

- [x] 2. インフラ層実装
- [x] 2.1 (P) yt-dlpクライアント
  - yt-dlpのPython APIを使い、チャンネルURLからflat extractionで全動画IDリストを取得する機能を実装する
  - 個別動画のメタデータ（タイトル、配信日時、動画の長さ）を取得する機能を実装する
  - 動画の字幕をjson3形式で取得し、タイムスタンプ付きの構造化データとしてパースする機能を実装する。手動字幕を優先し、自動生成字幕にフォールバックする
  - 設定にCookieファイルパスが指定されている場合はyt-dlpのcookiefileオプションに渡し、メンバー限定動画にアクセスできるようにする
  - 字幕が存在しない動画の場合はNoneを返し、呼び出し元でスキップ判定できるようにする
  - 動画本体のダウンロードは一切行わないことを保証する（skip_download設定）
  - _Requirements: 1.2, 1.3, 1.4, 1.5, 1.6, 5.2, 5.3_

- [x] 2.2 (P) LLMクライアント
  - Anthropic Python SDKを使い、字幕テキストを送信して話題セグメントのリスト（開始時刻、終了時刻、要約テキスト）を返すClaude API呼び出しを実装する
  - Structured Outputsを使用し、出力が常に期待するJSONスキーマに準拠することを保証する
  - セグメンテーション用のシステムプロンプトを設計する。日本語の配信字幕を話題の切れ目で分割し、各セグメントに簡潔な要約をつける指示を含める
  - 使用モデル名はConfigから取得し、デフォルトはClaude Haiku 4.5とする
  - APIエラー時は適切な例外を発生させ、呼び出し元でリトライやスキップを制御できるようにする
  - _Requirements: 3.1_

- [x] 2.3 (P) エンベディングプロバイダー
  - EmbeddingProvider Protocolに準拠した実装を作成し、テキストリストを受け取ってベクトルリストを返す
  - デフォルト実装としてOpenAI text-embedding-3-smallを使用する。APIキーはConfigから取得する
  - ベクトルの次元数をプロパティとして公開し、DB側のベクトルテーブル作成時に参照できるようにする
  - APIエラー時は適切な例外を発生させる
  - _Requirements: 3.3, 4.1_

- [x] 3. コアサービス実装
- [x] 3.1 チャンネル管理サービス
  - チャンネルURLまたはIDを受け取り、yt-dlpクライアントでチャンネル名を解決してDBに登録する機能を実装する
  - 登録済みチャンネルの一覧を、各チャンネルの同期済み動画数と最終同期日時とともに返す機能を実装する
  - 指定チャンネルの同期済み動画一覧を、タイトル・配信日時とともに返す機能を実装する
  - 既に登録済みのチャンネルが再度登録された場合は重複を通知する
  - _Requirements: 1.1, 2.4, 2.5_

- [x] 3.2 (P) 話題セグメンテーションサービス
  - 字幕テキストを受け取り、LLMクライアントに送信して話題セグメントを取得し、DBに保存する一連の処理を実装する
  - 取得したセグメントの要約テキストをエンベディングプロバイダーでベクトル化し、sqlite-vecテーブルに保存する
  - 4時間を超える長時間配信の字幕は、45分チャンク・5分オーバーラップで分割してLLMに送信し、結果をマージする処理を実装する
  - 指定動画のセグメント一覧を時系列順に返す機能を実装する
  - _Requirements: 3.1, 3.2, 3.3, 3.4_

- [x] 3.3 (P) ハイブリッド横断検索サービス
  - 検索クエリに対してFTS5キーワード検索とベクトル意味検索を並行実行する機能を実装する
  - FTS5検索はtrigramインデックスに対するクエリを実行し、字幕行から該当セグメントを特定する
  - ベクトル検索はクエリテキストをエンベディングプロバイダーでベクトル化し、sqlite-vecでKNN検索を行う
  - 両方の結果をマージし、スコアリング・重複排除を行って関連度順にランキングする
  - 各検索結果に動画タイトル、チャンネル名、開始・終了時刻、要約テキスト、タイムスタンプ付きYouTube URL（`https://youtube.com/watch?v=xxx&t=123`形式）を含める
  - 検索結果が0件の場合は空リストを返す
  - _Requirements: 4.1, 4.2, 4.3, 4.4_

- [x] 3.4 差分同期サービス
  - 全登録チャンネルに対して差分同期を実行するオーケストレーション処理を実装する
  - チャンネルごとにflat extractionで全動画IDを取得し、DB上の既存IDと比較して新規動画を特定する
  - 新規動画ごとに「字幕取得→DB保存→セグメンテーション→ベクトル化→インデックス登録」のパイプラインを実行する
  - 個別動画の処理が失敗しても他の動画の処理を続行する（部分的失敗の許容）
  - 同期完了後に結果サマリー（取得済み件数、新規取得件数、スキップ件数、エラー件数とその詳細）を返す
  - チャンネルの最終同期日時を更新する
  - _Requirements: 1.2, 1.3, 1.4, 1.5, 1.6, 1.7_

- [x] 4. CLI統合
- [x] 4.1 channelとsyncコマンド
  - `kirinuki channel add <URL>` コマンドを実装し、チャンネル登録サービスを呼び出して結果を表示する
  - `kirinuki channel list` コマンドを実装し、登録チャンネル一覧を見やすいテーブル形式で表示する
  - `kirinuki channel videos <チャンネルID>` コマンドを実装し、チャンネル内の動画一覧を表示する
  - `kirinuki sync` コマンドを実装し、差分同期サービスを呼び出して進捗と結果を表示する
  - 各コマンドに`--help`オプションでの説明表示を含める
  - _Requirements: 1.1, 1.7, 2.4, 2.5, 6.1, 6.3_

- [x] 4.2 (P) searchとsegmentsコマンド
  - `kirinuki search <クエリ>` コマンドを実装し、横断検索サービスを呼び出して結果を見やすく表示する。タイムスタンプ付きYouTube URLをクリック可能な形で出力する
  - `kirinuki segments <動画ID>` コマンドを実装し、指定動画のセグメント一覧を時系列順に表示する
  - 検索結果が0件の場合は「該当なし」のメッセージを表示する
  - `--limit`オプションで検索結果の最大件数を指定可能にする
  - _Requirements: 3.4, 4.1, 4.2, 4.3, 4.4, 6.1, 6.3_

- [x] 5. テストと品質保証
- [x] 5.1 データベース統合テスト
  - インメモリSQLiteを使用してスキーマ作成、CRUD操作、FTS5検索、ベクトル検索が正しく動作することを検証する
  - FTS5 trigramで日本語テキストの部分一致検索が機能することを確認する
  - sqlite-vecによるKNN検索が正しい距離順で結果を返すことを確認する
  - マイグレーションのバージョン管理が正しく動作することを確認する
  - _Requirements: 2.1, 2.2, 2.3_

- [x] 5.2 同期パイプラインE2Eテスト
  - yt-dlpクライアント、LLMクライアント、エンベディングプロバイダーをモック化した上で、チャンネル登録→同期→検索の完全なワークフローを検証する
  - 差分同期が正しく動作すること（2回目の同期で既存動画がスキップされること）を確認する
  - 字幕なし動画のスキップとエラー集計が正しく行われることを確認する
  - 検索結果にタイムスタンプ付きYouTube URLが正しく含まれることを確認する
  - _Requirements: 1.2, 1.6, 1.7, 4.3_
