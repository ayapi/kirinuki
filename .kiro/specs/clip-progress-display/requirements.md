# Requirements Document

## Introduction
clipコマンド実行時、現在は `[1/3] ダウンロード・切り抜き中...` という簡素なメッセージのみが表示される。
ダウンロードと再エンコードにはそれなりの時間がかかるため、ユーザーが処理の進行状況を把握できるよう、
各ステップの進捗をリアルタイムに表示する機能を追加する。

マルチクリップ時は、一番上の行に全体進捗 `[n/total]` を表示し、
その次の行から処理中のクリップの数だけ行を確保して、それぞれ単一クリップと同じ進捗情報を表示する。

## Requirements

### Requirement 1: 単一クリップの進捗表示
**Objective:** ユーザーとして、clipコマンド実行中にダウンロードと再エンコードの進捗をリアルタイムに確認したい。処理が進んでいるのか止まっているのか判断できるようにするため。

#### Acceptance Criteria
1. While yt-dlpによるダウンロードが進行中, the CLIは ダウンロード進捗をリアルタイムに表示する（進捗率・ダウンロード済みサイズ・速度・残り時間のうち、yt-dlpから取得可能な情報を含む）
2. While ffmpegによる再エンコードが進行中, the CLIは 再エンコード中であることを示すステータスを表示する
3. When ダウンロードまたは再エンコードが完了した, the CLIは 完了を示すメッセージを表示する
4. While 進捗が更新される, the CLIは 同一行を上書き更新し、ターミナルをスクロールさせない

### Requirement 2: マルチクリップ時の進捗レイアウト
**Objective:** ユーザーとして、複数区間を一括切り抜きする際に、全体進捗と各クリップの個別進捗を同時に確認したい。

#### Acceptance Criteria
1. When 複数の時間範囲が指定された, the CLIは 1行目に全体進捗を `[完了数/全体数]` 形式で表示する
2. While 複数クリップが並列処理されている, the CLIは 全体進捗行の下に処理中のクリップ数だけ行を表示し、各行にRequirement 1と同じ個別進捗情報を表示する
3. When 個々のクリップ処理が完了した, the CLIは 全体進捗の完了数を更新し、完了したクリップの行を処理中の表示から除去する
4. When すべてのクリップ処理が完了した, the CLIは 成功・失敗件数のサマリーと各クリップの結果を表示する（既存動作を維持）

### Requirement 3: 並列実行時の表示整合性
**Objective:** ユーザーとして、並列処理中に進捗表示が崩れず常に読みやすい状態であってほしい。

#### Acceptance Criteria
1. While 複数クリップが並列実行されている, the CLIは 進捗メッセージが互いに上書きしたり混在したりすることを防ぐ
2. The CLIは 並列実行時にスレッドセーフな方法で進捗表示を更新する
3. While 進捗表示が更新される, the CLIは ちらつきや描画崩れなく表示を行う
