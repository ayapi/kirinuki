# Implementation Plan

- [x] 1. セグメンテーション未完了動画の検出と字幕データ取得機能を追加する
- [x] 1.1 (P) セグメンテーション未完了の動画IDを取得する機能を実装する
  - チャンネルIDを指定して、動画は保存済みだがセグメントが存在しない動画IDの一覧を返す
  - セグメント済みの動画は結果に含まれないことを確認するテストを書く
  - 動画・セグメントの混在状態（一部のみセグメント済み）で正しく検出されることを確認する
  - 動画が存在しないチャンネルIDでは空リストを返すことを確認する
  - _Requirements: 1.1, 3.2_

- [x] 1.2 (P) DB保存済みの字幕エントリーを取得する機能を実装する
  - 動画IDを指定して、保存済みの字幕行を時系列順のエントリーリストとして返す
  - 字幕が複数行ある場合に開始時刻順で返されることを確認するテストを書く
  - 字幕が存在しない動画IDでは空リストを返すことを確認する
  - _Requirements: 1.1, 3.3_

- [x] 2. (P) SyncResultモデルに再試行結果フィールドを追加する
  - セグメンテーション再試行の成功件数と失敗件数を保持するフィールドを追加する
  - デフォルト値0で既存コードに影響がないことを確認するテストを書く
  - sync_allでの結果集約時に再試行フィールドも加算されるようにする
  - _Requirements: 2.1, 2.2_

- [x] 3. セグメンテーション再試行をsync処理に組み込む
  - 新規動画処理完了後に、セグメンテーション未完了の既存動画を検出して再試行する
  - DBから字幕データと動画の長さを取得し、既存のセグメンテーション処理に渡す
  - 再試行の成功・失敗をそれぞれカウントし、SyncResultに反映する
  - 各動画の再試行は独立して実行し、一つの失敗が他の動画の再試行を妨げないようにする
  - 再試行失敗時はログに警告を記録し、セグメントを保存しないことで次回syncの再試行対象として残す
  - 字幕データが空の動画はスキップする
  - YouTubeへのAPI呼び出しが発生しないことをテストで確認する
  - セグメンテーション済み動画が再処理されないことをテストで確認する
  - _Requirements: 1.1, 1.2, 1.3, 3.1, 3.2, 3.3_

- [x] 4. sync完了サマリーにセグメンテーション再試行の結果を表示する
  - 再試行が発生した場合のみ、成功・失敗件数をサマリーに追加表示する
  - 再試行がない場合は従来のサマリー表示のみとし、表示が変わらないことを確認する
  - _Requirements: 2.1, 2.2_
