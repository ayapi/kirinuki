# Implementation Plan

- [x] 1. データモデルとDBクライアントの拡張
- [x] 1.1 (P) SuggestOptionsとSuggestResultのフィールド追加
  - `SuggestOptions`に動画IDリストのオプショナルフィールドを追加し、チャンネルIDもオプショナルに変更する
  - `SuggestResult`に警告メッセージリストのフィールドを追加する（デフォルト空リスト）
  - 既存のフィールド（count, threshold等）は変更しない
  - _Requirements: 1.1, 1.2, 1.3, 1.4, 2.2_

- [x] 1.2 (P) DatabaseClientに動画ID指定取得メソッドを追加
  - 指定された動画IDリストに対応する動画情報（ID, タイトル, 公開日, 再生時間）を取得するメソッドを追加する
  - 返り値は既存の最新動画取得メソッドと同じ辞書リスト形式にする
  - 存在しないIDは結果に含まれず、呼び出し元で差分判定できるようにする
  - _Requirements: 1.1, 2.1_

- [x] 2. SuggestServiceにvideo_idフィルタリングロジックを追加
  - [x] 2.1 動画取得の分岐ロジックを実装する
  - video_ids指定時はDBから指定IDの動画を取得し、未指定時は従来の最新N件取得を使用する分岐を追加する
  - video_ids指定時はチャンネルID・count引数を無視する
  - video_ids未指定時はチャンネルIDの存在チェックと最新動画取得の既存ロジックを維持する
  - 存在しない動画IDを検出し、警告メッセージを生成して結果に含める（`search`コマンドと同一の警告フォーマット）
  - 有効な動画が1件もない場合はNoArchivesErrorを送出する
  - _Requirements: 1.1, 1.3, 1.4, 2.1, 2.2, 2.3_

- [x] 3. CLIにvideo-idオプションを追加し統合する
- [x] 3.1 suggestコマンドに--video-idオプションを追加する
  - `search`コマンドと同じ形式で、複数回指定可能な`--video-id`オプションを追加する
  - video-id指定時はチャンネル引数の解決をスキップし、チャンネル未指定でもエラーにしない
  - video-id未指定時は従来通りチャンネル引数を必須として解決する
  - オプション値をSuggestOptionsのvideo_idsフィールドに渡す
  - _Requirements: 1.1, 1.2, 1.3, 1.4_

- [x] 3.2 警告と進捗メッセージの表示を実装する
  - サービスから返された警告メッセージをstderrに出力する
  - video-id指定時にも対象動画のタイトルと公開日を進捗メッセージとして表示する（既存の`_status`関数を活用）
  - JSON出力モード時は進捗メッセージをstderrのみに出力する（既存動作と一致）
  - _Requirements: 2.2, 3.1, 3.2_

- [x] 3.3 既存のTUI・JSON出力フローとの連携を確認する
  - video-id指定とTUIモードの組み合わせで推薦結果の選択・切り抜き実行が正しく動作することを確認する
  - video-id指定とJSON出力の組み合わせで、既存のJSONスキーマを維持した出力が得られることを確認する
  - _Requirements: 4.1, 4.2, 5.1, 5.2_

- [x] 4. テストの実装
- [x] 4.1 (P) SuggestServiceのユニットテストを追加する
  - video_ids指定時に指定ID取得メソッドが呼ばれ、最新N件取得は呼ばれないことを検証する
  - video_ids未指定時に従来の最新N件取得が呼ばれることを検証する
  - 一部の動画IDが存在しない場合に警告が設定され、存在する動画のみで推薦が実行されることを検証する
  - 全動画IDが存在しない場合にNoArchivesErrorが送出されることを検証する
  - _Requirements: 1.1, 1.3, 1.4, 2.1, 2.2, 2.3_

- [x] 4.2 (P) DatabaseClientのユニットテストを追加する
  - 指定IDの動画取得メソッドが存在するIDのみ返却し、不在IDを除外することを検証する
  - 返り値が既存の最新動画取得と同一のキー構造であることを検証する
  - _Requirements: 1.1, 2.1_

- [x] 4.3 CLIの統合テストを追加する
  - `--video-id`オプションが正しくパースされ、チャンネル引数なしで動作することを検証する
  - `--video-id` + `--json`の組み合わせで正しいJSON出力が得られることを検証する
  - _Requirements: 1.1, 1.2, 3.1, 5.1_
