# Implementation Plan

- [x] 1. 動画ID/URL統一解決関数の追加
- [x] 1.1 (P) `resolve_video_id()` を実装する
  - URLまたは動画IDの文字列を受け取り、動画IDを返す関数を既存のクリッピングユーティリティモジュールに追加する
  - 11文字の動画IDパターン（英数字、ハイフン、アンダースコア）に一致する場合はそのまま返す
  - それ以外の文字列は既存のURL解析関数に委譲してIDを抽出する
  - 両方に該当しない場合は無効URL/IDエラーを送出する
  - _Requirements: 1.2, 1.3, 3.2_

- [x] 1.2 (P) `resolve_video_id()` のユニットテストを作成する
  - YouTube URL入力（`youtube.com/watch?v=`, `youtu.be/`, `youtube.com/live/`）から正しく動画IDを返すことを検証する
  - 11文字の動画ID直接入力でそのまま返ることを検証する
  - 無効な文字列（短すぎる、不正な文字、空文字列）で例外が送出されることを検証する
  - _Requirements: 1.2, 1.3, 3.2_

- [x] 2. 切り抜きオーケストレーションサービスの実装
- [x] 2.1 `ClipService` を実装する
  - 動画ダウンローダーとffmpegクライアントを受け取り、切り抜きリクエストを処理するサービスを新規作成する
  - 処理フロー: ffmpeg存在確認 → 出力先親ディレクトリ確認 → 一時ディレクトリに動画DL → ffmpegで指定区間切り出し → 結果返却
  - 一時ディレクトリは標準ライブラリのコンテキストマネージャで管理し、正常・異常問わず確実にクリーンアップする
  - 進捗通知はコールバック関数で受け取り、DL開始・切り出し開始のタイミングで呼び出す
  - 出力先の親ディレクトリが存在しない場合はファイル不在エラーを送出する
  - Cookie認証は既存のダウンローダーに委譲する（追加実装不要）
  - _Requirements: 2.1, 2.2, 2.3, 2.4, 3.3, 3.4, 4.1, 4.2_

- [x] 2.2 `ClipService` のユニットテストを作成する
  - 正常系: モックしたダウンローダーとffmpegクライアントで、DL→切り出し→結果返却の一連フローが正しく動作することを検証する
  - 異常系: DL失敗時に一時ディレクトリがクリーンアップされることを検証する
  - 異常系: ffmpeg切り出し失敗時に一時ディレクトリがクリーンアップされることを検証する
  - 出力先の親ディレクトリが存在しない場合にエラーが送出されることを検証する
  - 進捗コールバックが正しい順序で呼ばれることを検証する
  - _Requirements: 2.1, 2.2, 2.3, 3.3, 4.1, 4.2_

- [x] 3. CLIコマンド定義とシステム統合
- [x] 3.1 `kirinuki clip` コマンドを実装する
  - 4つの位置引数（video, start, end, output）を受け取るCLIコマンドを定義する
  - 引数パース: `video` は `resolve_video_id()` でID解決、`start`/`end` は既存の時刻パース関数で秒数変換する
  - 変換した値から切り抜きリクエストモデルを構築し、`ClipService.execute()` に委譲する
  - 進捗コールバックとして `click.echo` を渡す
  - 完了時に出力ファイルパスと切り抜き時間範囲を表示する
  - 各種ドメインエラー（無効URL、時間範囲不正、ffmpeg未検出、DL失敗、切り出し失敗、認証要求、出力先不在）をキャッチし、ユーザー向けエラーメッセージを表示して終了する
  - メインCLIグループにコマンドを登録し、アプリコンテキスト生成時に `ClipService` を組み込む
  - _Requirements: 1.1, 1.4, 1.5, 3.1, 3.5, 4.3_

- [x] 3.2 CLIコマンドのインテグレーションテストを作成する
  - clickの `CliRunner` を使用して引数パースの正常系を検証する
  - 無効な引数（不正なURL、時間範囲逆転等）でエラーメッセージが表示されることを検証する
  - ffmpeg未検出時のエラーメッセージ表示を検証する
  - _Requirements: 1.1, 3.1, 3.2, 3.4, 3.5_
