# Implementation Plan

- [x] 1. MultiClipRequestにmargin_secondsフィールドを追加する
  - 切り抜きリクエストモデルに`margin_seconds`フィールドをデフォルト`0.0`で追加する
  - 負の値を拒否するバリデーションを追加する（`margin_seconds >= 0`）
  - デフォルト`0.0`によりCLI clipコマンド等の既存コードは変更不要で後方互換性を保持する
  - _Requirements: 2.1, 2.2, 3.2_

- [x] 2. ClipServiceにマージン定数とマージン適用ロジックを実装する
  - モジュールレベルにデフォルトマージン秒数の定数（5.0秒）を定義する
  - 切り抜き処理の中でリクエストのマージン値を参照し、開始時刻をマージン分早め、終了時刻をマージン分遅らせてダウンロードに渡す
  - 開始時刻が負になる場合は0にクランプする
  - 終了時刻が動画長を超える場合はyt-dlpの既定動作に委ねる（エラーにしない）
  - 切り抜き結果には元の時間範囲（マージン適用前）を保持する
  - _Requirements: 1.1, 1.2, 1.3, 3.1_

- [x] 3. TUI execute_clipsでマージン付きリクエストを構築する
  - TUIフローの切り抜きリクエスト構築時にデフォルトマージン定数をインポートし、マージン秒数として設定する
  - _Requirements: 1.1_

- [x] 4. テスト
- [x] 4.1 (P) ClipServiceのマージン適用ユニットテストを実装する
  - マージン有り（5.0秒）でdownload_sectionに渡される開始・終了時刻が正しく拡張されることを検証する
  - マージン無し（0.0秒）で元の開始・終了時刻がそのまま渡されることを検証する
  - 開始時刻が5秒未満の区間でマージン適用時に0にクランプされることを検証する
  - _Requirements: 1.1, 1.2, 1.3, 2.1, 2.2_

- [x] 4.2 (P) MultiClipRequestのmargin_secondsバリデーションテストを実装する
  - デフォルト値が0.0であることを検証する
  - 正の値が受け入れられることを検証する
  - 負の値がバリデーションエラーになることを検証する
  - _Requirements: 3.2_
