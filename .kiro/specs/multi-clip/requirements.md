# Requirements Document

## Introduction

`kirinuki clip` コマンドを再設計し、同一動画の複数箇所を一度のダウンロードで効率的に切り出す機能を実装する。
現在の実装では1回のコマンド実行で1箇所しか切り抜けず、同じ動画から複数箇所を切り抜く場合は毎回動画をダウンロードし直す必要がある。
本機能により、カンマ区切りで複数の時間範囲を指定し、動画を1回だけダウンロードして順次切り出せるようにする。
後方互換性は不要とし、CLIインターフェースを刷新する。

## Requirements

### Requirement 1: CLIインターフェース

**Objective:** ユーザーとして、動画URL・出力ファイル名・時間範囲をシンプルに指定して切り抜きを実行したい。そうすれば、直感的で素早く操作できる。

#### Acceptance Criteria
1. The CLI shall `kirinuki clip <video> <filename> <time_ranges>` の形式でコマンドを受け付ける
2. The CLI shall 時間範囲をカンマ区切りで受け付ける（例: `18:03-19:31,21:31-23:20`）
3. The CLI shall 時間範囲が1つの場合もカンマなしで受け付ける（例: `18:03-19:31`）
4. The CLI shall `--output-dir` オプションで出力先ディレクトリを指定できる

### Requirement 2: 複数時間範囲による一括切り出し

**Objective:** ユーザーとして、1回のコマンド実行で同じ動画の複数箇所を切り出したい。そうすれば、動画のダウンロードが1回で済み、時間効率が良い。

#### Acceptance Criteria
1. When 複数の時間範囲が指定された場合, the ClipService shall 動画を1回だけダウンロードし、すべての時間範囲の切り出しを完了してからダウンロード動画を削除する
2. When 単一の時間範囲が指定された場合, the ClipService shall 1つの切り抜きファイルのみを生成する
3. If 指定された時間範囲のいずれかが不正（開始時刻 >= 終了時刻、負の値、フォーマット不正など）である場合, the CLI shall 不正な範囲を特定するエラーメッセージを表示し、処理を実行しない

### Requirement 3: 出力ファイル名と連番

**Objective:** ユーザーとして、出力ファイル名を指定し、複数切り抜き時には自動的に連番が付いてほしい。そうすれば、ファイルが整理しやすい。

#### Acceptance Criteria
1. When 複数の時間範囲が指定された場合, the CLI shall ファイル名に連番を付与する（例: `動画.mp4` → `動画1.mp4`, `動画2.mp4`）
2. When 単一の時間範囲が指定された場合, the CLI shall 連番を付与せずそのままのファイル名を使用する（例: `動画.mp4`）
3. The CLI shall ファイル名の拡張子の前に連番を挿入する

### Requirement 4: 出力先ディレクトリの設定

**Objective:** ユーザーとして、切り抜き動画の出力先ディレクトリを設定ファイルであらかじめ指定しておきたい。そうすれば、毎回出力先を指定する手間が省ける。

#### Acceptance Criteria
1. The AppConfig shall 出力先ディレクトリの設定項目 `output_dir` を持ち、デフォルト値は `~/.kirinuki/output` とする
2. When ユーザーがCLI引数 `--output-dir` で出力先を指定した場合, the CLI shall 設定ファイルの値よりCLI引数を優先する
3. When 出力先ディレクトリが存在しない場合, the CLI shall ディレクトリを自動的に作成する

### Requirement 5: エラーハンドリングと進捗表示

**Objective:** ユーザーとして、複数箇所の切り抜き処理中に進捗状況がわかり、一部失敗しても残りが続行されてほしい。そうすれば、長時間の処理でも安心して待てる。

#### Acceptance Criteria
1. While 複数の切り抜きを処理中, the CLI shall 現在の処理番号と総数を表示する（例: `[2/5] 切り抜き中...`）
2. If 個別の切り抜き（ffmpeg処理）が失敗した場合, the ClipService shall 失敗した範囲のエラーを記録し、残りの切り抜きを続行する
3. When すべての切り抜き処理が完了した場合, the CLI shall 成功数・失敗数のサマリーと各ファイルの出力パスを表示する
4. If 動画のダウンロード自体が失敗した場合, the CLI shall すべての切り抜き処理をスキップし、ダウンロードエラーを表示する
