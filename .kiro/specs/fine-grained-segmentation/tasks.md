# Implementation Plan

- [x] 1. LLMプロンプトの刷新と再分割API追加
- [x] 1.1 (P) セグメンテーション用システムプロンプトを細粒度分割向けに全面改訂する
  - 「短すぎるセグメントは統合」の指示を削除し、サブトピック単位での分割を指示する
  - 分割基準（サブトピック変化、Q&A、コメント反応、場面転換、作業フェーズ切替）を明記する
  - 品質基準（識別可能な話題のみ、フィラー・相槌は隣接に含める、最小長制約なし）を追加する
  - 要約フォーマットを階層的にする（「【大分類】サブトピック」形式、40文字以内）
  - _Requirements: 2.1, 2.2, 2.3, 3.1, 3.2, 3.3_

- [x] 1.2 (P) 長大セグメント再分割用のプロンプトとAPIメソッドを追加する
  - 親セグメントの要約をコンテキストとして注入する再分割専用プロンプトを定義する
  - 「最低2つ以上に分割」「階層的要約」を指示する
  - 既存のanalyze_topicsと同じJSONパースパターンで再分割メソッドを実装する
  - _Requirements: 1.3, 3.3_

- [x] 2. データベース層にセグメント管理機能を追加
- [x] 2.1 (P) セグメントの一括削除機能を実装する
  - 指定された動画のセグメントとベクトルデータを削除する（FK参照順序に注意）
  - 削除件数を返却し、他動画のデータに影響しないことを保証する
  - _Requirements: 5.1_

- [x] 2.2 (P) セグメント済み動画ID一覧の取得機能を実装する
  - segmentsテーブルからDISTINCTでvideo_idの一覧を返却する
  - _Requirements: 5.3_

- [x] 3. セグメンテーションサービスのコアロジックを改修する
- [x] 3.1 チャンク分割の定数を調整する
  - チャンクサイズを45分から20分に短縮する
  - オーバーラップを5分から3分に調整する
  - 既存の重複排除ロジックは維持する
  - _Requirements: 4.1, 4.2, 4.3_

- [x] 3.2 セグメント最大長制御と再分割処理を実装する
  - セグメンテーション処理にmax_segment_msパラメータを追加する（デフォルト5分）
  - LLM呼び出し後、最大長を超えるセグメントを検出し、該当区間の字幕を再度LLMに送信して再分割する
  - 再分割は1パスのみ実行し、2つ以上に分割できた場合のみ採用する
  - 再分割失敗時・1件以下返却時は元セグメントを保持する
  - _Requirements: 1.1, 1.2, 1.3, 1.4_

- [x] 3.3 既存セグメントの再セグメンテーション機能を実装する
  - 指定動画の既存セグメントを削除し、DB内の字幕データから再セグメンテーションを実行する
  - YouTube APIの再呼び出しは不要（DB内字幕を再利用）
  - _Requirements: 5.1_

- [x] 4. 同期サービスにセグメンテーションパラメータを伝搬する
- [x] 4.1 max_segment_msパラメータを同期処理の全メソッドに伝搬する
  - sync_all → sync_channel → _sync_single_video / _retry_segmentation の呼び出しチェーン全体にパラメータを追加する
  - デフォルト値は300,000（5分）を設定する
  - _Requirements: 1.2, 6.1, 6.2_

- [x] 5. CLIにセグメンテーション制御オプションとサブコマンドを追加する
- [x] 5.1 syncコマンドにセグメント最大長オプションを追加する
  - --max-segment-msオプション（デフォルト300000、整数型）を追加する
  - sync_service.sync_allにパラメータを伝搬する
  - _Requirements: 6.1, 6.2_

- [x] 5.2 resegmentサブコマンドを新設する
  - --video-id指定時は単一動画のみ再セグメンテーションする
  - --max-segment-msでセグメント最大長を指定可能にする
  - オプションなしでセグメント済み全動画を処理し、進捗表示（N/total）を出力する
  - 個別動画のエラーは表示して次の動画に進む
  - _Requirements: 5.1, 5.2, 5.3, 5.4, 6.1, 6.2_

- [x] 6. テストを追加する
- [x] 6.1 (P) LLMクライアントの再分割メソッドのテストを追加する
  - 再分割セグメントの返却、親要約のプロンプト注入、空入力、JSONパース失敗の各ケースを検証する
  - _Requirements: 1.3, 3.3_

- [x] 6.2 (P) セグメンテーションサービスの再分割・再セグメンテーションのテストを追加する
  - 超過セグメントの再分割、非超過セグメント保持、再分割失敗フォールバック、1件返却フォールバックを検証する
  - 既存セグメント削除→再生成、字幕なし、動画不存在の再セグメンテーションケースを検証する
  - _Requirements: 1.1, 1.3, 1.4, 5.1_

- [x] 6.3 (P) データベース層のセグメント管理テストを追加する
  - セグメント+ベクトル削除、0件削除、他動画への影響なしを検証する
  - セグメント済みID一覧取得と空テーブルのケースを検証する
  - _Requirements: 5.1, 5.3_
